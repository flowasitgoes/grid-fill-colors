## 完成畫面倒數與平滑跳轉需求備註

- 問題：檢查成功後立即跳轉下一步，玩家無法停留欣賞成果。
- 目標：增加「恭喜」停留與倒數提示，給玩家在原畫面緩衝時間。
- 功能需求：
  1. 成功時顯示慶祝訊息與倒數提示（預設 3 秒）。
  2. UI 提供「X」按鈕取消自動跳轉。
  3. 倒數計時只在完成視圖顯示期間啟動；玩家重玩或離開時需清除計時。
  4. 無障礙：顯示明確提示（例如「倒數 3 秒後查看成績，按下關閉以停留」）。
- 倒數流程說明：
  - 啟動條件：`gameWon === true` 且 overlay 顯示。
  - UI 顯示：「倒數 3 秒後查看成績」文字 + 取消按鈕。
  - 玩家按「X」則停止倒數，提示改為「倒數已取消，可以慢慢欣賞」。
  - 若倒數結束且未取消，觸發 `next` 事件推進。
## 完成畫面倒數與平滑跳轉需求備註

- 問題：檢查成功後立即跳轉下一步，玩家無法停留欣賞成果。
- 目標：增加「恭喜」停留與倒數提示，給玩家在原畫面緩衝時間。
- 功能需求：
  1. 成功時顯示慶祝訊息與倒數提示（預設 3 秒）。
  2. UI 提供「X」按鈕取消自動跳轉。
  3. 倒數計時只在完成視圖顯示期間啟動；玩家重玩或離開時需清除計時。
  4. 無障礙：顯示明確訊息（例如「倒數 3 秒後查看成績，按下關閉以停留」）。
- 技術策略：
  - 在 `CompletionOverlayComponent` 新增倒數 state、使用 `setInterval` 逐秒更新。
  - 可透過 `@Input()` 控制是否啟用倒數；完成成功時啟動，失敗時不倒數。
  - 當倒數結束且未被取消，觸發 `next` 事件；若玩家按 X，則停止倒數並顯示「已取消」狀態。
- 待處理：
  - 提供倒數數字顯示和進度條（可先以數字為主）。
  - 計時器在 overlay 關閉或元件銷毀時需清除，避免 memory leak。
