## 完成畫面 UX 草稿

### 目標
- 給予玩家清楚的完成回饋（成功 / 失敗），加強成就感與再戰動機。
- 呈現最關鍵的完成統計，協助玩家理解表現並鼓勵重玩。
- 提供明確的行動選項：重玩、前往下一關、回主選單；預留分享/下載入口。

### 視覺結構
1. **背景覆蓋層**：半透明深色漸層加上柔和噴光，焦點聚焦在彈窗。
2. **結果標題區**  
   - 成功：金色煙火、`🎉 完成啦！`字樣。  
   - 失敗：暖色燈條 + `再試一次吧`，避免過度負面。  
   - 加入動態線條／粒子，配合音效。
3. **統計卡片**（使用分段卡片呈現）  
   - 完成耗時（含分鐘/秒）  
   - 使用提示次數  
   - 填色總步數（含擦除次數）  
   - 連續正確填色最佳紀錄（若無則顯示「-」）  
   - 完成效率評等（依耗時與提示數組合，分為 A / B / C）
4. **成果縮圖**  
   - 展示玩家完成後的 5x5 網格截圖（直接使用當前網格資料渲染）。  
   - 預留「下載 PNG / 分享」按鈕位置。
5. **CTA 區塊**  
   - `重玩本關`（主要按鈕，深紫漸層）  
   - `挑戰下一關`（若尚有下一關，次要按鈕；否則顯示「已完成所有關卡！」並顯示返回）  
   - `返回選單`（文字按鈕）  
   - 成功時另顯示「分享成就」預留按鈕（灰色 Disabled 狀態）

### 音效與動態
- 成功時觸發：歡呼 + 煙火層（延長到 1.2s）、場景環境音淡出 0.6s。
- 失敗時觸發：柔和下降音 + UI 提示音，避免強烈失落。
- CTA 滑入動畫 180ms、統計卡片依序浮現 120ms 延遲。

### 無障礙考量
- 對比度 ≥ 4.5:1，按鈕提供聚焦樣式。
- 統計項目使用對齊列表，搭配 icon 與文字（讀屏朗讀順序按照標題 → 統計 → CTA）。
- 完成耗時提供文字敘述（例如：「耗時 1 分 24 秒」）。

### 後續擴充預留
- 分享/下載實作時可直接使用成果縮圖區塊的按鈕。
- 評等規則可改成根據難度加權，先在 `GameService` 中保留計算函式。
- 失敗視圖可加入「顯示錯誤格」按鈕接到提示機制。
